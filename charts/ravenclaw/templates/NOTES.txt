Thank you for installing {{ .Chart.Name }} {{ .Chart.AppVersion }}!

Your release is named {{ .Release.Name }} and is deployed in the {{ .Release.Namespace }} namespace.

**Getting Started:**

1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "ravenclaw.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
  NOTE: It may take a few minutes for the LoadBalancer IP to be available.
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "ravenclaw.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else if contains "ClusterIP" .Values.service.type }}
  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "ravenclaw.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  export CONTAINER_PORT=$(kubectl get pod --namespace {{ .Release.Namespace }} $POD_NAME -o jsonpath="{.spec.containers[0].ports[0].containerPort}")
  echo "Visit http://127.0.0.1:8080 to use your application"
  kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 8080:$CONTAINER_PORT
{{- end }}

2. Check the health endpoint:
   curl http://localhost:8080/health

3. Check readiness:
   curl http://localhost:8080/ready

4. View metrics:
   curl http://localhost:8080/metrics

**Configuration:**

The RavenClaw configuration is stored in a ConfigMap named {{ include "ravenclaw.fullname" . }}-config.
You can modify it and restart the deployment:

  kubectl edit configmap {{ include "ravenclaw.fullname" . }}-config -n {{ .Release.Namespace }}
  kubectl rollout restart deployment {{ include "ravenclaw.fullname" . }} -n {{ .Release.Namespace }}

**Secrets:**

API keys should be provided via Helm values (--set secrets.LITELLM_API_KEY=...) or
created externally and referenced via existingSecrets.
